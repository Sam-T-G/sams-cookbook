# Voice guide

The house writing voice for every README, doc, and recipe. The `/draft-recipe-prose` skill enforces it.
Write like a sharp teammate, not a consultant.

## Rules

- We-voice by default. Use "I" only for a personal judgment in a reflection.
- No em dashes. Restructure, use a comma or semicolon or parentheses, or split the sentence. This is the
  loudest single tell, so it is a hard rule. (source: ChatGPT-polished-writing studies)
- No AI-tell vocabulary: delve, leverage, showcase, robust, seamless, tapestry, at its core, navigate the
  complexities, in essence, it is worth noting, moreover, additionally.
- No "this isn't X, it's Y" contrast scaffolds as a default sentence shape.
- Straight quotes, not curly.
- Captions are one to three sentences. State what the code does and why it matters.
- A reflection is prose, three to five sentences, and carries a difficulty encountered plus a forward note.
  It is a journal entry, not a success report.
- Every rule or claim cites a source. Version facts reference `versions.lock`, not raw build numbers.
- Plain headers beat clever ones. "What the screen does" beats "User interface overview."
- Comment why, not what. The code already says what.

## Self-check before handing back

Grep your own draft for em dashes and for the banned words. Read the reflection: does it admit a difficulty
and point forward, or is it a sterile "everything went well"? Fix it if it is sterile.
